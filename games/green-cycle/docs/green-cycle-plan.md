## 1. 技术选型与理由

| 层面          | 选型                                            | 理由                                                                                                                              |
| ------------- | ----------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **渲染**      | HTML5 Canvas 2D（单 `<canvas>`）                | 单文件、零外部渲染依赖、跨平台；像素风 2D 场景完全够用。若 50 波后期同屏 200+ 单位出现瓶颈，再迁移到 WebGL/PixiJS，前期保持简单。 |
| **状态管理**  | 集中式 `GameState` + 轻量 ECS-lite              | 塔防实体多、系统独立（移动、战斗、经济、波次），ECS-lite 便于后续新增塔/敌人类型而不修改核心逻辑。                                |
| **开发语言**  | TypeScript                                      | 类型安全、重构友好；便于维护塔/敌人/配方等大量数据表。                                                                            |
| **构建工具**  | Vite + `vite-plugin-singlefile`                 | 开发阶段模块化 + HMR，构建产物自动内联为单个 `index.html`，符合“单文件优先”要求。                                                 |
| **UI 交互层** | HTML/CSS Overlay + Pointer Events               | 建造面板、英雄属性页、波次提示等用 DOM 实现更快、更易响应式；Canvas 只负责游戏世界渲染。                                          |
| **音频**      | Web Audio API + 程序化音效（zzfx 或自写 Synth） | 不依赖外部音频文件，直接生成 8-bit 像素音效，适合单文件部署。BGM 可选项外链或程序生成循环。                                       |
| **输入**      | Pointer Events（鼠标/触摸）+ 键盘快捷键         | 同时支持 PC 与移动端；快捷键用于加速波次、技能释放、快速出售。                                                                    |
| **多人合作**  | MVP 不做实时联网                                | 先实现单人 + 本地热座/AI 补位 + 异步排行榜。实时联机（WebSocket/RTC）作为后续扩展，避免阻塞核心玩法验证。                         |
| **持久化**    | `localStorage`                                  | 满足存档、排行榜、设置需求；无需后端。                                                                                            |

## 4. 关键实现方案

### 4.1 循环路径与敌人移动

- **路径表示**：`Path` 类维护 `points: Vec2[]`、`segmentLengths`、`totalLength`。
- **循环参数**：敌人持有 `pathProgress ∈ [0, 1]`，每帧更新：
  ```
  progress += speed * dt / totalLength
  if progress >= 1: progress -= 1   // 无限循环
  ```
- **位置插值**：根据 `progress` 找到对应线段，线性插值得到 `(x, y)`。
- **多圈结构**：路径点按“外圈 → 连接道 → 内圈 → 连接道 → 外圈”顺序首尾相连，形成闭合循环。
- **对象池**：敌人使用 `EnemyPool`，死亡时重置字段并回池，避免 GC 波动。
- **移动状态**：实际速度 = 基础速度 × (1 + 驱逐加速 - 减速 - 缓速)。状态叠加采用带持续时间的 `Buff/Debuff` 列表。

### 4.2 塔攻击与目标选择

- **塔数据字段**：位置、射程、攻速、攻击类型、基础伤害、目标策略、投射物速度、冷却时间。
- **索敌策略枚举**（每种塔可配置默认策略）：
  - `nearest`：射程内距离最近（多数成长塔默认）。
  - `first`：`pathProgress` 最大，最先漏怪。
  - `strongest`：血量最高（适合单体爆发）。
  - `weakest`：血量最低（适合补刀）。
  - `priority`：优先光环怪 / 飞行怪 / Boss。
- **攻击循环**：每帧 `cooldown -= dt`；若 `cooldown <= 0` 且找到目标，则生成 `Projectile` 或即时命中，并重置冷却。
- **伤害结算入口**：`CombatSystem.applyDamage(tower, enemy)`，后续接入护甲克制、暴击、光环倍率。
- **投射物**：记录起点/目标/速度；命中时结算；使用对象池；可简化成“无弹体即时伤害 + 视觉特效”。

### 4.3 护甲/攻击类型克制表

- **攻击类型**：普通、穿刺、魔法、混乱、神圣。
- **护甲类型**：物理、魔法、神圣、抗性。
- **建议伤害系数表**（需 PRD 最终确认）：

| 攻击 \ 护甲 | 物理 | 魔法 | 神圣 | 抗性 |
| ----------- | ---- | ---- | ---- | ---- |
| 普通        | 1.00 | 0.75 | 1.00 | 1.00 |
| 穿刺        | 1.50 | 0.50 | 0.75 | 1.00 |
| 魔法        | 0.75 | 1.50 | 1.00 | 0.50 |
| 混乱        | 1.00 | 1.00 | 1.00 | 1.00 |
| 神圣        | 1.00 | 1.00 | 1.50 | 1.00 |

- **UI 提示**：
  - 敌人头顶显示护甲颜色条（物理红、魔法蓝、神圣金、抗性紫）。
  - 鼠标悬停塔/敌人时显示“攻击类型 → 护甲类型”的克制关系。
  - 伤害跳字颜色：绿色=加成，白色=正常，红色=减益。

### 4.4 成长塔加点/技能系统

- **成长塔字段**：等级、当前经验、升级所需经验、属性点、技能点、力量/敏捷/智力、4 个技能槽。
- **升级规则**：
  - 击杀敌人获得经验；经验达到阈值升级。
  - 每升 1 级获得 1 点属性点；每升 5 级获得 1 点技能点。
- **属性影响**：
  - 力量：提升技能伤害 / 生命值。
  - 敏捷：提升攻速，存在上限（例如 300 敏捷达到面板攻速上限）。
  - 智力：提升技能触发概率 / 法强 / 回蓝。
- **技能系统**：
  - 每个成长塔有 4 个技能；开局选择 1 个起手技能。
  - 技能分 `active`（主动释放/自动释放）、`passive`（被动常驻）、`aura`（范围光环）。
  - 技能点可升级技能等级，最高 3–5 级。
- **UI**：选中成长塔打开英雄面板，显示经验条、属性条、加点按钮、技能图标与说明。

### 4.5 合成与配方系统

- **配方定义**：
  ```ts
  interface Recipe {
    id: string;
    name: string;
    ingredients: { towerId: string; level?: number }[];
    result: { towerId: string; level?: number };
    cost?: { gold?: number; wood?: number };
  }
  ```
- **合成触发**：
  1. 玩家选中多座塔或在合成面板点击配方。
  2. 系统扫描场上塔，匹配 `ingredients`。
  3. 扣除资源，移除原料塔，在指定位置生成 `result` 塔。
- **套装合成**：如“五虎将 → 魔神吕布”，需要 5 座特定塔同时存在；合成后合并为 1 座高级塔。
- **UI**：合成按钮根据当前场上塔动态高亮可用配方；鼠标悬停显示所需材料与结果。

### 4.6 资源与经济

- **三种核心资源**：金币、木材、人口。
- **收入**：
  - 击杀奖励：怪物死亡时即时掉落金币（主要来源）。
  - 波次奖励：每波结束按击杀率结算。
  - 经济波：每隔 5–7 波出现高奖励单位（兔子/飞艇）。
  - Boss 击杀：掉落木材。
- **支出**：
  - 造塔：消耗金币 + 人口。
  - 升级：消耗金币。
  - 购买人口：消耗木材。
  - 刷新商店/购买道具：消耗木材。
- **人口机制**：
  - 每座塔占用 1 点或多点人口。
  - 超过人口上限无法建造；UI 实时显示 `当前 / 最大`。
- **利息**：本版本不实现银行存款利息，符合原版与传统外传设定。

### 4.7 保存/加载（localStorage）

- **存档键名**：`greenCycle_v1`。
- **保存内容**：
  - 玩家设置（音量、画质、快捷键）。
  - 解锁进度（通关难度、无尽模式解锁）。
  - 排行榜（最高波次、无尽分数、PF 分）。
  - 当前对局快速保存（可选，用于单局暂停续玩）。
- **加载流程**：启动时读取存档；若版本不匹配则迁移或重置。
- **自动保存**：每波结束、游戏暂停、关键事件后异步写入。
- **清除**：设置页提供“重置所有存档”按钮。

## 5. 风险与简化建议

| 风险               | 影响                                           | 应对/简化建议                                                                                               |
| ------------------ | ---------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| **PRD 数值未定稿** | 塔参数、波次表、护甲系数缺失会导致反复返工     | 先确定一份“最小数值表”再进入 M3；用 Excel/TS 数据文件集中管理，便于热更新。                                 |
| **后期性能瓶颈**   | 50 波后期 150+ 敌人 + 大量投射物 + 光环计算    | 使用对象池、空间哈希/四叉树优化索敌、非关键系统每 N 帧更新、限制粒子数量、优先 Canvas 2D 批量绘制。         |
| **平衡性调参**     | 护甲克制、成长塔成长曲线、波次血量曲线相互耦合 | 建立“波次模拟器”脚本，自动跑多局并输出 DPS/漏怪率；先做简单/普通难度。                                      |
| **多人实时合作**   | 网络同步、作弊、断线重连复杂                   | MVP 不实现实时联机，改为本地热座或 AI 队友；联网作为后续版本。                                              |
| **美术资源**       | 像素美术若外包会拉长工期                       | 全部程序生成 16×16/24×24 像素精灵，用颜色区分护甲与塔类型；后期再替换美术。                                 |
| **版本机制过多**   | 海量版/外传 III 的套装、装备、宠物等机制庞大   | 第一版只保留原版 50 波 + 经典塔 + 3–5 个成长塔 + 2–3 个合成配方；飞行/隐身/魔免等特殊能力按优先级逐步加入。 |
