# 风格规范

> 校准基准：[`assets/towers/tower_arrow.svg`](file:///Users/doer/dev/yoo/games/green-cycle/assets/towers/tower_arrow.svg)

## 1. 视角约定

- **正交俯视**（orthographic top-down）：摄像机严格从正上方往下
- 视高 = 0，不存在近大远小
- 物体顶部轮廓 = 物体俯视剪影
- 厚度感通过 2~3 个层叠 `rect` 的明暗对比实现（顶面亮、侧面中、底面暗）
- **严禁 45° / 等距 / 透视**

## 2. 网格与对齐

- 主网格 4px（= 16×16 格 @ 64×64）
- 次网格 2px（仅在需要 8×8 半格微调时使用，但 4px 主轴仍需对齐）
- 所有 `x`、`y`、`width`、`height` 必须是 2 的倍数
- 跨格块体必须 4 对齐（不要 `x=13` 这种半格偏移）

## 3. 调色板规则

### 数量
- 主体 ≤ 8 色
- 简单图标 ≤ 4 色
- 每色至少出现 ≥ 2 个像素（孤立单像素易被压缩失真）

### 配色三段式（推荐）
| 段 | 用途 | 亮度 |
|---|---|---|
| Dark | 阴影 / 轮廓 | 20~35% |
| Base | 主体色 | 45~60% |
| Light | 高光 / 边缘 | 75~90% |

特殊色：
- **腐蚀/污渍**：在 Dark 上加 1~2 个对比色点（≤ 4 个像素）
- **辉光/魔法**：用 2 色（中等 + 高亮）做小范围点状放射

### 主题色板（常用 6 套）

| 主题 | Dark | Base | Light | Accent |
|---|---|---|---|---|
| toxic（毒/酸） | `#1a3d00` | `#4a8a00` | `#9acd32` | `#bfff00` |
| holy（圣/光） | `#8b6914` | `#ffd700` | `#fff8dc` | `#ffffff` |
| ice（冰/霜） | `#0d3b66` | `#1e90ff` | `#b0e0e6` | `#ffffff` |
| fire（火/熔） | `#5c1a00` | `#ff4500` | `#ffaa33` | `#ffe066` |
| dark（暗/死） | `#1a1a2e` | `#4a3b6b` | `#9b6b9b` | `#ff0040` |
| nature（自然） | `#2d4a1a` | `#5a8c3a` | `#a8d56a` | `#ffe066` |

可在主色板上自由微调，但**保持 Dark→Base→Light 三段亮度的对比度**。

## 4. 形状规范

- **圆角**：`rx` / `ry` ≤ 1（4px 块的 25%），或干脆 0
- **抗锯齿**：通过 `shape-rendering="crispEdges"` 强制像素对齐
- **镂空**：用第二个 `<rect>`（同背景色）覆盖，不使用 `mask`（兼容性差）
- **细线**：至少 2px 宽（1px 在小尺寸下不可见）
- **斜线/曲线**：用阶梯状 `<rect>` 拼出（pixel-perfect），不使用 `path` 曲线

## 5. 安全边

64×64 画布：
- 主体范围 6~58（占 81%）
- 边缘 0~6 / 58~64 是透明安全边

32×32 画布：
- 主体范围 3~29（占 81%）
- 边缘 0~3 / 29~32 是透明安全边

## 6. 禁止清单

- ❌ `linearGradient` / `radialGradient` / `pattern`
- ❌ `filter`（drop-shadow、blur、glow 等）
- ❌ `path` 曲线（用 rect 阶梯替代）
- ❌ `text` 元素（图标内不放文字）
- ❌ 多个图层混合模式 / `opacity < 1`（除非要刻意做半透效果）
- ❌ 45° 旋转 / skew 变换
- ❌ 圆形 `circle` 用作主体（用方块拼阶梯圆）
